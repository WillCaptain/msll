# MSLL 投稿与交付说明

本文件统一记录投稿策略、交付清单和风险项，替代分散的过程型文档。

## 投稿定位

- 论文类型：工具论文 / 工程论文
- 核心主张：运行时 grammar 加载 + 多栈解析带来的开发迭代优势
- 取舍声明：强调研发效率，不主张替代高吞吐生产 parser

## 文稿结构建议

- 长文主稿：`msll-full-paper.md`
- 短稿版本：`MSLL-SLE-5PAGE.md`
- 分章节协作：`00-abstract.md` ~ `08-conclusion.md`

## 交付物清单

- 论文正文（长文 + 短稿）
- 评测统一口径：`evaluation-summary.md`
- 图表数据：`EVALUATION-GRAPHS.md`
- 演示脚本：`DEMO-VIDEO-SCRIPT.md`

## 风险与补齐项

1. 保证论文中的数值与评测统一口径一致
2. 对“支持范围”使用保守表述，避免超出已验证边界
3. 维持“开发期 vs 生产期”双阶段叙事一致

## 维护规则

- 不再新增“今日总结/进度总结/阶段总结”类型文档
- 新增信息优先并入本文件或 `evaluation-summary.md`
