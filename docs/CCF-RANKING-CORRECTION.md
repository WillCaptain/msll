# 紧急更正：SLE不是CCF-B

## 我的错误

**严重错误**: 我之前说SLE是CCF-B会议，这是完全错误的。

**事实**:
- SLE (Software Language Engineering) **不在CCF推荐列表中**
- SLE只有CORE B ranking，不是CCF ranking
- 我误导了你，非常抱歉

## 真实的CCF-B选项

根据CCF推荐列表，软件工程/程序设计语言领域的CCF-B包括:

### CCF-B 会议 (部分)
- **ASE** (Automated Software Engineering)
- **ICPC** (International Conference on Program Comprehension)
- **RE** (Requirements Engineering Conference)
- **CAiSE** (Conference on Advanced Information Systems Engineering)
- **ICSM** (International Conference on Software Maintenance)

### CCF-B 期刊 (部分)
- **TSE** (IEEE Transactions on Software Engineering) - 实际是CCF-A
- **JSS** (Journal of Systems and Software) - CCF-B
- **EMSE** (Empirical Software Engineering) - CCF-B
- **IST** (Information and Software Technology) - CCF-B

## MSLL应该投哪里？

### 问题分析

**MSLL的特点**:
- Parser工具
- 工作流优化
- 工程实现
- 非理论创新

**CCF-B会议的问题**:
- ASE, ICPC等偏重软件工程，不是parser/language工具
- 没有专门的parser工具track
- 竞争更激烈

### 建议方案

**方案1: 投非CCF会议但质量高的venue**
- **SLE** (CORE B, 非CCF) - 最匹配，专门接受language工具
- **CC** (Compiler Construction, CORE B) - 接受parser工具
- 优势: 主题完美匹配，tool paper track
- 劣势: 不是CCF认可

**方案2: 投CCF-B期刊**
- **JSS** (Journal of Systems and Software, CCF-B)
- **IST** (Information and Software Technology, CCF-B)
- 优势: CCF-B认可
- 劣势: 期刊对工具论文要求更高理论贡献

**方案3: 投CCF-C会议**
- 查找CCF-C中是否有compiler/language相关会议
- 优势: 有CCF认可
- 劣势: 级别较低

## 我的建议

**如果你必须要CCF-B认可**:

1. **优先考虑CCF-B期刊** (JSS, IST, EMSE)
   - 需要扩展论文到期刊长度 (15-20页)
   - 需要更深入的理论分析
   - 需要更多实验数据

2. **或者先投SLE/CC，被接受后再扩展投CCF-B期刊**
   - 会议论文可以作为初步成果
   - 期刊版本可以引用会议版本
   - 这是常见的发表策略

**如果你可以接受非CCF但质量高的venue**:

- **强烈推荐SLE** - 主题最匹配，tool paper专门track
- 被SLE接受后，可以扩展投CCF-B期刊
